jsonargparse[all] 子依赖 jsonnet 0.22.0 是 C++ 扩展，编译需要 Python.h 头文件（python3-devel 包）。Dockerfile 未显式安装 python3-devel（因 miniconda 自带 Python.h），但 openEuler 官方 python3 不带 devel 头文件，新手按 Dockerfile 步骤时会卡在 jsonnet wheel build 失败。

遇到的问题 (4)

pip 默认清华源下载 pip-26.1.2 升级包返回 HTTP 403 Forbidden，无法升级 pip。

方案: 切换到阿里云源（mirrors.aliyun.com/pypi/simple）后升级 pip 成功。

来源: pip config set global.index-url https://mirrors.aliyun.com/pypi/simple

jsonnet 0.22.0 pyproject.toml 构建失败：fatal error: Python.h: No such file or directory。

方案: 通过 dnf install -y python3-devel 安装 Python 头文件后，jsonnet 0.22.0 C++ 扩展构建成功（编译产物 _jsonnet.abi3.so）。

来源: jsonnet 0.22.0 build log: Python.h 缺失

torchvision 默认解析 torch 2.12.1（最新），与项目要求的 torch==2.7.1 冲突。

方案: 显式 pin torchvision==0.22.1 / torchdata==0.11.2（这两个版本与 torch 2.7.1 兼容），避免解析到新版本。

来源: pip install torchvision (无版本约束时)

tests/ut 66 个测试文件中 35 个 collection error，全部因 import torch_npu / megatron.core / megatron.training 等 NPU 专属模块失败。

方案: 仅执行 31 个无 NPU 依赖的 CPU 可执行测试（test_aspect_ratio.py / test_ema.py / test_ut_demo.py），全部 31 个用例通过；其余 35 个标注 skipped（NPU 环境受限），最终 UT 状态为 partial_success。

来源: tests/ut/*/ 35 个测试文件 collection error
